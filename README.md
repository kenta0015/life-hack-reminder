# life-hack-reminder

## コンセプト

人生で大事なことほど忘れる。だから、忘れたときに思い出させる仕組みが必要。

これはToDo管理でも、名言配信でもなく、**自分専用のライフハック（言葉・コツ・型）を"思い出せる形で育てる"**アプリ。

## アプリの中身（何を扱うか）

ユーザーが保存するのは、基本的に「自分に効くもの」だけ。主に3種類（内部分類）：

1) Life Cards（モチベ・格言・決意）

画像＋短い言葉で構成される"刺さるカード"

通知をタップしたときに 画像と言葉をドーンと出して、気持ちや視点を上げる用途に向く

有名な格言でもいいが、最終的には 自分の言葉／自分が刺さった言葉が混ざるほど価値が上がる

2) Nudges（人生のコツ・注意喚起・ちょい修正）

短い"気をつけること"や"コツ"を入れる枠

例：慌ててる時ほど呼吸を深く、深呼吸で集中力アップ、会話で焦ったら一呼吸、など

モチベの演出より、日常で忘れがちなコツを思い出させる用途

3) Playbooks（状況別の型・攻略手順）

仕事、人間関係、学び、行動など、幅広い場面の「型」

Nudgesが"1行"だとしたら、Playbooksは状況→判断→行動の短い手順として保存するイメージ

「いろんな領域を入れたい」という要求をここで吸収できる

※この3つは"機能が増える"というより、同じ目的（思い出させる）に対する表現の違い。

## 配信（どう思い出させるか）

保存した内容は、アプリが定期的に通知で思い出させる

Life Cardsは特に「通知→タップ→画像＋言葉ドーン」が体験の中心になりやすい

頻度は多すぎるとノイズ化するので、週3回など低頻度が合う（ここは方針として）

反応で"育つ/消える"仕組み（重要な差別化）

通知で見せたあと（または時々）、アプリが短く聞く：

「この言葉（/コツ）は役に立ちましたか？」

ここでの反応が、次に出てくる頻度や生存に影響する。

## 反応の基本方針

YES（役に立った）
→ 次に思い出させるまでの間隔がどんどん長くなる
（暗記アプリの"間隔反復"に近い考え方。定着してきたら頻度を下げる）

NO（役に立たなかった / 今は違う）
→ **削除（または退役）**の対象になる
「今の自分に合ってないものは残さない」という思想を実装する

（任意）SKIP / Meh
→ 判断保留。頻度は維持または少し下げる、など

これが意味すること

溜めるほど偉いメモアプリではなく、
**"効くものだけが残っていく"**アプリになる

人生のステージが変わるほど、内容が自然に入れ替わり、
過去に縛られずにアップデートされる運用になる

## 10個に絞る思想（現時点のコンセプト要素）

リマインドできる数を10個程度に制限する案がある

目的：本当に大事なものに絞る／ノイズを防ぐ／ステージが変わったら入れ替えやすくする

「増やす」より「入れ替える」ことで、ユーザーが成長していく思想に合う

（※この"10"は、強い思想として採用する方向で話が進んでいる要素）

## ここまでの仕様を一言でまとめると

自分の格言・ライフハック・型を登録する

アプリが定期的に思い出させる

「役に立った？」の反応で、

役に立つほど表示間隔が伸びる（定着）

役に立たないものは削除/退役（淘汰）

結果として、ユーザーの変化に合わせて
"今の自分に効くもの"だけが残る
